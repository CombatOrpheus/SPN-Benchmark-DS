## 2024-04-01 - Cythonizing Core Functions
**Learning:** When using Cython alongside Numba for optional acceleration, `cdef` functions in Cython (`.pyx`) cannot be directly imported or used by Python. Doing so causes an `ImportError`, triggering the Numba fallback and nullifying the optimization. Also, compiled binaries (`.so`) and intermediate C files (`.c`, `.cpp`) generated by Cython build steps should never be committed to source control.
**Action:** Ensure that any Cython function meant to be called from Python is declared with `def` or `cpdef`, not `cdef`. Additionally, always verify that `*.so`, `*.c`, and `*.cpp` are added to `.gitignore` when adding Cython extensions to a project, and delete any generated binaries before committing.
